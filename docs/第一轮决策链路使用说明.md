# 第一轮决策链路使用说明

> 本说明用于在 IDEA / PyCharm / 终端中手动运行第一轮“天才交易员”决策链路。  
> 当前阶段只做本地辅助判断，不自动交易，不连接券商，不写数据库。

---

## 1. 当前能做什么

当前链路：

```text
stock_data_<code>.json
  ↓
time_context
  ↓
strategy_snapshot
  ↓
decision_result
  ↓
markdown report
```

支持：

- 单票买入预评估。
- 单票持仓卖出/持有预评估。
- 缺数据时自动降级。
- 缺持仓信息时输出 `PRE_EVALUATION`。
- 生成 JSON 和 Markdown 报告。

不支持：

- 自动下单。
- 模拟盘成交。
- 数据库保存。
- Web 界面。
- 全市场扫描。

## 4. 持仓卖出/持有评估

### 4.1 缺持仓信息时

只基于当前数据做预评估：

```powershell
python .\build_decision.py 002563 --task sell
```

典型输出：

```text
action: PRE_EVALUATION confidence=MEDIUM
```

含义：

- 系统可以看当前价、20 日线、60 日线、近 20 日高低点。
- 但因为缺成本价、仓位、买入逻辑、证伪点，只能预评估。

### 4.2 提供完整持仓信息

```powershell
python .\build_decision.py 002563 --task sell `
  --avg-cost 5.10 `
  --position-pct 12 `
  --total-quantity 2000 `
  --available-quantity 2000 `
  --buy-logic "middle-term recovery" `
  --invalidation-point "break ma60 and profit weakens" `
  --holding-period middle
```

参数说明：

| 参数 | 含义 |
|---|---|
| `--avg-cost` | 成本价 |
| `--position-pct` | 当前仓位百分比 |
| `--total-quantity` | 总持仓数量 |
| `--available-quantity` | 今日可卖数量 |
| `--buy-logic` | 当初买入逻辑 |
| `--invalidation-point` | 原证伪点 |
| `--holding-period` | `short` / `middle` / `long` |

## 5. 买入评估

```powershell
python .\build_decision.py 002563 --task buy
```

当前第一轮策略很保守，通常只会输出：

```text
WATCH_SMALL
WAIT
DO_NOT_BUY
DATA_BLOCKED
```

第一轮不会轻易输出强 `BUY`。

原因：

- 还没有完整资金系统。
- 还没有完整策略验证。
- 还没有模拟盘。
- 估值和公告原文仍有缺口。

## 6. 输出文件在哪里

运行后会生成三类文件。

### 6.1 策略快照

```text
data/strategy_snapshots/
```

示例：

```text
strategy_snapshot_002563_holding_review_20260704T110428857691.json
```

这是策略引擎真正读取的数据。

### 6.2 决策结果

```text
data/decision_results/
```

示例：

```text
decision_result_002563_holding_review_20260704T110428857691.json
```

这是机器可读结果。

### 6.3 Markdown 报告

```text
data/reports/
```

示例：

```text
decision_report_002563_holding_review_20260704T110428857691.md
```

这是你主要看的文件。

### 6.4 自动清理旧输出

默认每只股票、每类输出最多保留最近 5 个文件。

例如同一只股票多次运行后，只保留最近 5 个：

```text
data/strategy_snapshots/strategy_snapshot_002563_*.json
data/decision_results/decision_result_002563_*.json
data/reports/decision_report_002563_*.md
```

可以通过参数调整：

```powershell
python .\build_decision.py 002563 --task sell --keep-outputs 3
```

如果想暂时不清理，可以给一个很大的数字：

```powershell
python .\build_decision.py 002563 --task sell --keep-outputs 999
```

## 8. action 含义

买入评估：

| action | 含义 |
|---|---|
| `BUY` | 买入。第一轮基本不会给 |
| `WATCH_SMALL` | 观察仓 / 小仓试错 |
| `WAIT` | 观望 |
| `DO_NOT_BUY` | 不买入 |
| `DATA_BLOCKED` | 数据不足，暂停评估 |

持仓评估：

| action | 含义 |
|---|---|
| `HOLD` | 持有 |
| `REDUCE_HALF` | 减仓一半 |
| `REDUCE_TO_WATCH` | 减到观察仓 |
| `CLEAR` | 清仓 |
| `NO_SELL_T_PLUS` | 触发卖出但 T+1 或可卖数量限制 |
| `PRE_EVALUATION` | 缺持仓信息，只能预评估 |
| `DATA_BLOCKED` | 数据不足，暂停评估 |

## 9. 当前注意事项

- 当前是第一轮最小闭环，不是完整交易系统。
- `build_decision.py` 不会自动抓数据，必须先有 `stock_data_<code>.json`。
- 卖出评估如果不提供成本、仓位、买入逻辑、证伪点，默认只会给 `PRE_EVALUATION`。
- 买入评估不会轻易给强 `BUY`。
- 当前报告还比较简单，后续会继续优化中文输出。
- 当前输出目录已加入 `.gitignore`，不会默认提交生成的结果文件。
