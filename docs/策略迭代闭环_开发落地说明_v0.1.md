# 策略迭代闭环开发落地说明 v0.1

> 本文档用于指导第一版“策略调参记录”功能。  
> 目标是把历史回放和模拟盘结果沉淀成可追踪记录，不做自动调参，不替代人工复盘。

---

## 1. 核心定位

`v0.1` 已经在跑通交易辅助流程。

下一步需要一个最小闭环：

```text
跑历史回放 / 模拟盘
  -> 记录结果
  -> 写下策略假设和改动
  -> 标记问题
  -> 形成下一次修改方向
```

第一版只解决一件事：

```text
每次策略实验都有记录，后续能比较。
```

---

data/paper_trading/trades.jsonl

## 3. 输出

第一版输出：

```text
data/strategy_iterations/strategy_tuning_records.jsonl
data/reports/strategy_iteration_report.md
```

这些是本地复盘记录，不建议提交 Git。

## 5. 命令设计

记录一次历史回放：

```powershell
python .\strategy_iteration.py record --source-type replay --source-path .\data\replay\replay_xxx --hypothesis "突破买入是否有效" --rule-changes "提高买入确认门槛" --next-action "扩大样本再跑一年"
```

记录一次模拟盘：

```powershell
python .\strategy_iteration.py record --source-type paper --account paper_default --hypothesis "盘中触发是否减少漏买" --next-action "继续观察两周"
```

查看最近记录：

```powershell
python .\strategy_iteration.py list --limit 10
```

重生成汇总报告：

```powershell
python .\strategy_iteration.py report
```

---

## 6. 自动识别的问题

第一版自动识别：

- 没有交易。
- 总收益为负。
- 跑输基准。
- 最大回撤过大。
- 胜率过低。
- 交易过少导致样本不足。
- 阻塞原因过多。

自动识别只做提醒，不自动修改策略。

---

## 7. 边界

第一版不做：

- 自动调参。
- 参数网格搜索。
- 机器学习优化。
- 根据一次回放直接改策略。
- 自动实盘切换。

必须坚持：

```text
先记录
再比较
再人工判断
最后改规则
```

这个功能的价值不是让策略立刻变好，而是让每一次策略修改都有证据链。
