# 第四轮工作流使用说明

> 本文档说明如何在 IDEA 终端里使用 09 第一版工作流功能。  
> 当前工作流只做本地编排，不做自动定时、不连接券商、不自动实盘下单。

---

## 1. 当前能力

第四轮代码支持：

- 手动执行盘前工作流。
- 手动执行盘中触发扫描。
- 手动执行盘后工作流。
- 手动执行非交易研究扫描。
- 记录每次工作流运行状态。
- 盘前自动执行模拟盘换日和 T+1 解锁。
- 盘前批量生成第一轮买入决策。
- 盘前扫描当前模拟盘持仓，生成持仓复核决策。
- 盘前调用第三轮组合计划，生成买入候选排序和下单意图。
- 盘前输出触发价列表。
- 盘中读取触发价列表和当前行情，生成触发事件和中文提醒报告。
- 盘后生成账户快照、持仓快照、模拟盘报告和盘后工作流报告。

当前不支持：

- 自动定时调度。
- 自动盘中实时行情轮询。
- 消息推送。
- 自动确认成交。
- 实盘券商下单。
- 完整节假日日历。

## 2. 前置条件

先初始化模拟账户：

```powershell
python .\paper_trading.py init --account paper_default --cash 100000
```

确保要扫描的股票已有数据底稿：

```text
data/stock_json/stock_data_002563.json
data/stock_json/stock_data_600398.json
data/stock_json/stock_data_601138.json
```

如果没有数据，先抓取：

```powershell
python .\stock_decision_data.py 002563 --period middle
```

## 3. 盘前工作流

示例：

```powershell
python .\workflow.py pre-market --account paper_default --trade-date 2026-07-06 --symbols 002563,600398,601138
```

作用：

- 校验 `trade-date` 是否是工作日交易候选日。
- 执行模拟盘 `rollover`。
- 处理 T+1 可卖数量解锁。
- 对指定股票批量生成买入评估。
- 对当前持仓生成卖出/持有复核。
- 调用组合计划，输出可确认买入意图。
- 生成盘前计划和中文报告。

输出：

```text
data/workflows/workflow_runs.jsonl
data/workflows/pre_market_plan_<account>_<trade_date>.json
data/workflows/trigger_price_list_<account>_<trade_date>.json
data/reports/pre_market_report_<account>_<trade_date>.md
data/reports/allocation_report_<account>_<trade_date>.md
```

只扫描前 N 个本地数据文件：

```powershell
python .\workflow.py pre-market --account paper_default --trade-date 2026-07-06 --limit 3
```

使用自选股文件：

```powershell
python .\workflow.py pre-market --account paper_default --trade-date 2026-07-06 --watchlist .\data\watchlist.txt
```

`watchlist.txt` 可以每行一个股票代码：

```text
002563
600398
601138
```

## 4. 盘后工作流

示例：

```powershell
python .\workflow.py post-market --account paper_default --trade-date 2026-07-06
```

作用：

- 生成账户快照。
- 生成持仓快照。
- 更新模拟盘账户市值。
- 生成模拟盘报告。
- 生成盘后工作流报告。

输出：

```text
data/portfolio/account_snapshots.jsonl
data/portfolio/position_snapshots.jsonl
data/reports/paper_report_<account>_<trade_date>.md
data/reports/post_market_report_<account>_<trade_date>.md
```

同一账户同一交易日重复运行时，会保留最新快照，避免重复快照污染绩效统计。

## 5. 盘中触发扫描

示例：

```powershell
python .\workflow.py intraday --account paper_default --trade-date 2026-07-06
```

作用：

- 读取当天盘前计划。
- 读取当天触发价列表。
- 读取 `data/stock_json/stock_data_<code>.json` 当前价格。
- 判断买入突破、止损、跌破 MA20、跌破 MA60、跌破近 20 日低点。
- 检查行情日期是否等于扫描交易日。
- 检查停牌、涨跌停、持仓和 T+1 可卖数量。
- 写入触发事件和中文报告。

输出：

```text
data/intraday/trigger_events.jsonl
data/intraday/intraday_scans.jsonl
data/reports/intraday_report_<account>_<trade_date>.md
```

只扫描指定股票：

```powershell
python .\workflow.py intraday --account paper_default --trade-date 2026-07-06 --symbols 002563,600398
```

指定计划文件：

```powershell
python .\workflow.py intraday --account paper_default --trade-date 2026-07-06 --plan-path .\data\workflows\pre_market_plan_paper_default_2026-07-06.json --trigger-path .\data\workflows\trigger_price_list_paper_default_2026-07-06.json
```

周末或非盘中调试：

```powershell
python .\workflow.py intraday --account paper_default --trade-date 2026-07-06 --allow-non-trading
```

注意：

- 第一版盘中扫描只提醒，不生成成交。
- `execution_allowed` 固定为 `false`。
- 行情 `trade_date` 与扫描 `trade-date` 不一致时，会阻塞买卖触发。
- 同一账户、交易日、股票、事件类型、`decision_id` 已触发过时，不会重复写入。
- 真正模拟成交仍必须走人工确认命令。

## 6. 非交易研究

示例：

```powershell
python .\workflow.py research --symbols 002563,600398,601138
```

作用：

- 周末、晚上、节假日可以整理候选。
- 可以批量生成第一轮研究报告。
- 不执行 `rollover`。
- 不生成模拟成交。
- 不生成可执行下单计划。

输出：

```text
data/reports/research_report_<date>.md
```

## 7. 时间注意事项

时间是交易系统的硬约束。

第一版时间规则：

- `pre-market` 和 `post-market` 的 `--trade-date` 必须是工作日交易候选日。
- 当前版本只识别周一到周五，不含完整 A 股节假日历。
- 如果今天是周末，比如 2026-07-04 是周六，不能把当天当作可成交交易日。
- 周末可以为下一个交易候选日生成研究或盘前准备，但第一轮决策里的 `time_context.session_name` 仍会如实标注当前不是交易日。
- 盘前可以使用上一交易日收盘数据做当日计划，此时 `trade_date` 是目标交易日，`source_quote_trade_date` 是数据来源交易日。
- 非交易时段生成的计划不能直接当成成交指令。
- `intraday` 默认只允许集合竞价、上午交易和下午交易时段扫描；`--allow-non-trading` 只用于调试。

真正模拟成交仍必须单独执行：

```powershell
python .\paper_trading.py apply <decision_path> --account paper_default --confirm
```

## 8. 常见问题

### 8.1 为什么盘前没有可确认买入计划

常见原因：

```text
NON_TRADING_DAY
DATA_BLOCKED
PRICE_MISSING
PRICE_TIME_MISMATCH
DAILY_ROLLOVER_MISSING
NO_ELIGIBLE_CANDIDATE
CASH_NOT_ENOUGH_FOR_ONE_LOT
```

尤其在周末或行情数据不是目标交易日时，系统会保守地只记录研究结果。

### 8.2 为什么盘中触发了但不直接成交

因为当前目标是先验证策略、数据、风控和模拟盘链路。

交易动作必须经过：

```text
decision_result
  -> risk_check
  -> allocation_plan
  -> order_intent
  -> 人工确认
  -> paper_trading apply --confirm
```

### 8.3 盘中为什么没有触发

常见原因：

```text
NOT_INTRADAY_TRADE_SESSION
PRICE_TIME_MISMATCH
PRICE_MISSING
NO_READY_BUY_INTENT
NO_POSITION
NO_AVAILABLE_QUANTITY
SUSPENDED
LIMIT_UP_RISK
LIMIT_DOWN_RISK
```

如果今天是周末或晚上，默认会阻塞。调试可以加：

```powershell
python .\workflow.py intraday --account paper_default --trade-date 2026-07-06 --allow-non-trading
```

### 8.4 股票很多时怎么跑

第一版建议先小样本：

```powershell
python .\workflow.py pre-market --account paper_default --trade-date 2026-07-06 --limit 5
```

等策略稳定后，再扩大自选股池。
