# 第二轮模拟盘使用说明

> 本文档说明如何在 IDEA 终端里使用 04-05 第一版模拟盘功能。  
> 当前模拟盘只做本地纸面验证，不连接券商，不做实盘下单。

---

## 1. 当前能力

第二轮代码支持：

- 初始化模拟账户。
- 盘前换日 `rollover`。
- 按 T+1 解锁可卖数量。
- 读取第一轮 `decision_result`。
- 记录 `paper_signal`。
- 在 `--confirm` 后生成模拟订单和模拟成交。
- 维护现金、持仓、成本、可卖数量、锁定数量。
- 生成账户快照、持仓快照和中文模拟盘报告。

当前不支持：

- 实盘下单。
- 券商账户同步。
- 严谨历史回测。
- 分钟级撮合。
- 多账户组合风控。

---

## 2. 初始化模拟账户

在项目根目录执行：

```powershell
python .\paper_trading.py init --account paper_default --cash 100000
```

默认会生成：

```text
data/portfolio/accounts.json
data/portfolio/positions.json
data/portfolio/cash_ledger.jsonl
```

如果账户已存在，会提示 `account already exists`，不要重复初始化同名账户。

---

## 3. 盘前换日

每个交易日开始前先执行：

```powershell
python .\paper_trading.py rollover --account paper_default --trade-date 2026-07-06
```

作用：

- 处理到期 T+1 解锁。
- 重置 `today_buy_used`。
- 重置 `today_sell_amount`。
- 设置账户当前交易日。

注意：

- `trade-date` 必须是交易日。
- 当前版本只识别工作日，不含完整节假日历。
- 没有执行当日 `rollover` 时，`--confirm` 不允许生成模拟成交。

---

## 4. 生成第一轮决策

例如：

```powershell
python .\build_decision.py 002563 --task buy
```

或持仓评估：

```powershell
python .\build_decision.py 002563 --task sell --avg-cost 5.10 --position-pct 12 --total-quantity 2000 --available-quantity 2000 --buy-logic "中线修复" --invalidation-point "跌破60日线且业绩转弱"
```

会生成：

```text
data/strategy_snapshots/
data/decision_results/
data/reports/
```

---

## 5. 应用到模拟盘

只记录信号，不成交：

```powershell
python .\paper_trading.py apply .\data\decision_results\decision_result_002563_buy_evaluation_xxx.json --account paper_default
```

人工确认后，允许生成模拟订单和成交：

```powershell
python .\paper_trading.py apply .\data\decision_results\decision_result_002563_buy_evaluation_xxx.json --account paper_default --confirm
```

规则：

- `WAIT`、`DO_NOT_BUY`、`HOLD`、`PRE_EVALUATION` 只记录信号。
- `DATA_BLOCKED`、`NO_SELL_T_PLUS` 只记录阻塞信号。
- 周末、节假日、`NON_TRADING` 决策不能生成模拟成交。
- 未执行当日 `rollover` 时不能生成模拟成交。
- 买入必须是 100 股整数倍。
- 非清仓卖出必须按 100 股整数倍。
- 清仓允许卖出不足 100 股尾数。

---

## 6. 生成每日快照和报告

```powershell
python .\paper_trading.py snapshot --account paper_default
```

输出：

```text
data/portfolio/account_snapshots.jsonl
data/portfolio/position_snapshots.jsonl
data/reports/paper_report_paper_default_<trade_date>.md
```

即使当天没有交易，只要仍有持仓，也应该生成每日快照。

---

## 7. 输出文件说明

模拟盘输出：

```text
data/paper_trading/signals.jsonl
data/paper_trading/orders.jsonl
data/paper_trading/trades.jsonl
```

账户和持仓输出：

```text
data/portfolio/accounts.json
data/portfolio/positions.json
data/portfolio/position_locks.jsonl
data/portfolio/cash_ledger.jsonl
data/portfolio/position_ledger.jsonl
data/portfolio/account_snapshots.jsonl
data/portfolio/position_snapshots.jsonl
```

这些文件是本地模拟盘状态，不建议提交到 Git。

## 8. 常见问题

### 8.1 为什么周六不能模拟成交

因为当前决策的 `time_context.session_name = NON_TRADING`。

周末生成的报告只能作为计划参考，不能当作真实成交。

### 8.2 为什么需要先 rollover

因为 A 股有 T+1，系统必须在每日开始前处理可卖数量解锁，并重置当日买入额度。

### 8.3 为什么 `PRE_EVALUATION` 不会卖出

`PRE_EVALUATION` 表示缺少成本、仓位、原买入逻辑或证伪点，只能预评估，不能生成可执行卖出订单。

### 8.4 模拟盘结果能证明策略赚钱吗

不能。模拟盘只能验证链路和初步观察策略表现。  
全流程稳定后，需要继续做历史回放，用半年或一年的历史交易日快速验证策略。

如果要把一段模拟盘表现沉淀成调参记录，可以执行：

```powershell
python .\strategy_iteration.py record --source-type paper --account paper_default --hypothesis "观察当前策略模拟盘表现" --next-action "继续观察或回放验证"
```
