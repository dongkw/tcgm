# 08 数据库开发落地说明 v0.1

> 本文档用于指导 SQLite 主账本第一版开发。  
> 优先级：先支撑现有功能，再保证后续容易迁移 PostgreSQL。

---

## 1. 第一版目标

第一版数据库不是重写业务系统。

目标是：

```text
现有 JSON/JSONL 继续可用
SQLite 能导入和查询核心账本
后续界面可以直接读 SQLite
未来迁 PostgreSQL 时不大改业务模块
```

---

## 2. 技术边界

第一版使用：

```text
sqlite3 标准库
应用生成字符串 ID
SQL migration 文件
repository 层访问数据库
JSON 原始文件继续保留
```

第一版不使用：

```text
PostgreSQL 服务
SQLAlchemy
Redis
任务队列
多用户权限系统
自动实盘下单
```

后续如果代码复杂度上升，再考虑 SQLAlchemy 或轻量 ORM。

## 5. 开发顺序

### 5.1 初始化数据库

实现：

```powershell
python .\database.py init
```

要求：

- 创建 `data/ai_trader.sqlite3`。
- 执行 migration。
- 写入 `schema_migrations`。
- 重复执行不破坏已有数据。

### 5.2 导入现有 JSON/JSONL

实现：

```powershell
python .\database.py import-json
```

导入范围：

- 模拟盘账户、持仓、流水、快照。
- paper signals/orders/trades。
- strategy snapshots。
- decision results。
- risk checks、allocation plans、order intents。
- workflow runs。
- intraday scans、trigger events。
- replay runs、daily records、performance metrics。
- strategy tuning records。
- reports 文件索引。

要求：

- 可重复导入。
- 按业务 ID upsert。
- 每次导入写 `import_batches`。
- 导入失败不删除源文件。

### 5.3 对账校验

实现：

```powershell
python .\database.py validate
```

至少校验：

- 账户资产 = 现金 + 持仓市值。
- 持仓锁定数量 = 未释放锁汇总。
- 成交引用的订单存在。
- 信号引用的决策存在。
- 回放绩效能找到回放记录。
- 盘中触发能找到扫描记录。

### 5.4 查询摘要

实现：

```powershell
python .\database.py summary
```

输出：

- 账户数。
- 当前持仓数。
- 成交数。
- 决策数。
- 回放次数。
- 策略调参记录数。
- 最近报告。

### 5.5 备份

实现：

```powershell
python .\database.py backup
```

输出：

```text
data/backups/ai_trader_YYYYMMDD_HHMMSS.sqlite3
```

## 6. PostgreSQL 迁移约束

开发时必须遵守：

- 不在业务模块里直接写 SQL。
- 不依赖 SQLite `rowid`。
- 不用 `AUTOINCREMENT` 做业务主键。
- 不用 SQLite JSON1 做核心查询。
- 不用 SQLite FTS 做第一版功能。
- 所有 ID 由应用生成。
- 所有 JSON 字段以 `_json` 结尾。
- 所有状态使用字符串枚举。
- 所有数据库写入走 repository。

后续迁 PostgreSQL 时，主要替换：

```text
connection
migrations
repository SQL dialect
```

不改：

```text
paper_trading
workflow
historical_replay
strategy_iteration
```

## 7. 验收标准

第一版完成后应满足：

- `init` 可重复执行。
- `import-json` 能把现有数据导入 SQLite。
- `validate` 能发现基础账本不一致。
- `summary` 能给后续界面提供首页数据。
- `backup` 能生成数据库备份。
- `.sqlite3` 和备份目录不提交 Git。
- 后续迁 PostgreSQL 不需要推翻表结构。

## 8. 当前代码实现范围

已实现：

- `database.py init`：执行 `migrations/sqlite/*.sql`，可重复运行。
- `database.py import-json`：把现有 JSON/JSONL 账本导入 SQLite，写入 `import_batches` 和 `file_artifacts`。
- `database.py summary`：输出账户、持仓、成交、决策、回放、报告等摘要。
- `database.py validate`：做基础对账、引用完整性和 JSON 载荷检查。
- `database.py reconcile`：检查当前还存在的决策快照、决策结果和报告文件是否已经索引到 SQLite。
- `database.py backup`：用 SQLite backup API 生成一致性备份。

第一版仍然保留 JSON/JSONL 为原始事实源，SQLite 先作为查询索引和界面数据源。当前 `build_decision.py` 已经对 `strategy_snapshot`、`decision_result` 和报告索引做自动同步；资金、持仓、成交和 T+1 解锁暂时仍以文件账本为准。
