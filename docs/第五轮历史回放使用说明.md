# 第五轮历史回放使用说明

> 本文档说明如何在 IDEA 终端里使用第一版 `REPLAY_LITE` 历史回放。  
> 当前历史回放用于快速验证策略链路和明显问题，不是严谨专业回测。
> `v0.1` 的目标是跑通流程；后续真正的核心是用回放和模拟盘结果持续修正交易策略。

---

## 1. 当前能力

第五轮代码支持：

- 读取历史日线 JSONL。
- 按交易日逐日回放。
- 用上一交易日收盘数据生成当日盘前决策。
- 复用第一轮策略快照和决策结果。
- 复用第三轮风控和组合构建。
- 复用第二轮模拟盘账户、持仓、现金、T+1 和成本模型。
- 用当日开盘价模拟成交。
- 用当日收盘价估值生成每日快照。
- 输出每日回放记录、绩效指标和中文报告。

当前不支持：

- 自动抓取历史日线。
- 严格点时财报和公告。
- 分钟级或 tick 级撮合。
- 全市场批量回测。
- 自动参数优化。
- 图形化收益曲线。

## 2. 准备历史日线

默认读取目录：

```text
data/historical/daily_bars/
```

每只股票一个文件：

```text
data/historical/daily_bars/002563.jsonl
```

每行一条交易日行情：

```json
{"symbol":"002563","trade_date":"2025-01-02","open":5.12,"high":5.30,"low":5.05,"close":5.22,"pre_close":5.10,"volume":12345600,"amount":65000000,"pct_change":2.35,"is_suspended":false,"is_limit_up":false,"is_limit_down":false}
```

必需字段：

```text
symbol
trade_date
open
high
low
close
```

建议字段：

```text
pre_close
volume
amount
pct_change
is_suspended
is_limit_up
is_limit_down
```

注意：

- 第一版不会自动下载历史数据。
- 价格复权口径必须一致，建议先用前复权。
- 回放至少需要 60 个以上历史交易日，策略里的 MA60 才能正常计算。

## 3. 运行回放

单票回放：

```powershell
python .\historical_replay.py run --symbols 002563 --start 2025-01-01 --end 2025-12-31 --cash 100000
```

指定历史日线目录：

```powershell
python .\historical_replay.py run --symbols 002563 --start 2025-01-01 --end 2025-12-31 --cash 100000 --bar-dir data/historical/daily_bars
```

多票回放：

```powershell
python .\historical_replay.py run --symbols 002563,600398 --start 2025-01-01 --end 2025-12-31 --cash 100000
```

限制候选数量：

```powershell
python .\historical_replay.py run --symbols 002563,600398,601138 --start 2025-01-01 --end 2025-12-31 --cash 100000 --max-candidates 3
```

## 4. 输出目录

每次运行会生成独立目录：

```text
data/replay/<replay_id>/
```

主要文件：

```text
replay_config.json

daily_replay_records.jsonl
performance_metrics.json
replay_report.md

decision_results/
risk_checks.jsonl
allocation_plans.jsonl
order_intents.jsonl
paper_trading/
portfolio/
```

这些文件是本地回放结果，不建议提交 Git。

## 5. 怎么看结果

优先看：

```text
data/replay/<replay_id>/replay_report.md
data/replay/<replay_id>/performance_metrics.json
data/replay/<replay_id>/daily_replay_records.jsonl
```

重点指标：

- 总收益率。
- 最大回撤。
- 交易次数。
- 买入次数。
- 卖出次数。
- 基准收益。
- 超额收益。
- 每日阻塞原因。

注意：

```text
交易次数太少时，不能证明策略有效。
```

第一版更重要的是发现：

- 策略是否长期不交易。
- 是否频繁被数据阻塞。
- 是否买入后很快亏损。
- 是否因为 T+1、涨跌停、现金不足导致执行失败。
- 是否需要补历史数据字段。

## 6. 时间口径

第一版使用盘前口径：

```text
D-1 收盘数据
  ↓
D 日 08:45 生成盘前决策
  ↓
D 日开盘价模拟成交
  ↓
D 日收盘价估值
```

这意味着：

- 不允许用 D 日收盘数据决定 D 日开盘买卖。
- 不允许用当前最新财报回填过去。
- 不允许用未来最高价或最低价优化买卖。

`strategy_snapshot` 中：

```text
trade_date = D
source_quote_trade_date = D-1
session_name = PRE_MARKET
```

## 7. REPLAY_LITE 限制

`REPLAY_LITE` 没有严格点时财报和公告。

所以：

- 财报字段缺失不会伪造。
- 公告字段缺失不会伪造。
- 财务和公告相关判断不能证明有效。
- 当前主要验证价格、趋势、仓位、风控、T+1、手续费和滑点。

如果后续要验证完整基本面策略，需要进入 `PIT_STRICT` 数据体系。

## 8. 记录策略迭代

跑完一次历史回放后，建议立刻记录这次策略实验：

```powershell
python .\strategy_iteration.py record --source-type replay --source-path .\data\replay\replay_xxx --hypothesis "验证当前买入规则" --rule-changes "无，本次作为基线" --next-action "扩大样本继续回放"
```

输出：

```text
data/strategy_iterations/strategy_tuning_records.jsonl
data/reports/strategy_iteration_report.md
```

这个记录用于后续比较策略版本，不会自动修改策略。

## 9. 常见问题

### 9.1 为什么没有交易

常见原因：

```text
历史数据少于 60 个交易日，MA60 缺失
趋势规则没有触发
数据质量被阻塞
现金不足
涨停买入被拦截
策略当前只给 WAIT
```

### 9.2 为什么第一天没有决策

盘前回放需要上一交易日数据。

如果 `start` 是数据文件第一天，则当天没有 `D-1`，系统会跳过或记录阻塞。

### 9.3 结果赚钱能不能直接上实盘

不能。

第一版回放只说明这段历史样本下链路和规则表现如何。  
后续还要看样本外、不同年份、多只股票、最大回撤、交易成本和模拟盘连续表现。
